About Layers

在区块链领域中 Layer 是指区块链网络架构的不同层级.

为了提高系统的扩展性、效率和处理能力，区块链实现了模块化发展.

Layer 2（第二层）是指在区块链技术和网络协议中用于扩展基础区块链（Layer1）的解决方案。其目的是提高交易速度、降低交易费用，并增强网络的可扩展性和效率。.

代币标准

Token Standards

new standards

ERC-20

以太坊代币标准 / 开头为0x

最受欢迎和广泛使用的代币标准之一，BEP-20（开头为0x）和TRC-20（开头为T）都是它的模仿者。它们是智能合约的技术标准，通过使用虚拟机（EVM）发行代币，当各类钱包、交易所在对接合约的资产时，从这套合约标准中就可以知道它定义了哪些函数、事件，从而方便的进行对接.

ERC-721

NFT标准 / 开头为0x

这是首个非同质化代币（non-fungible token, NFT）协议，它旨在创建非同质化代币 ( NFT ) 并管理具有独特属性的虚拟资产。在ERC-721标准里，每个Token都拥有独立唯一的ID编号，具有不可取代性.

ERC-1155

混合标准 / 开头为0x

这一标准结合了 ERC-20 和 ERC-721的功能。它允许在一个智能合约中创建同质化和不可同质化代币。主要功能包括与不同类型代币进行批量交易、获取代币余额和元数据、真实性验证等.

old standards

Omni

比特币标准 / 2012 - 2014

OMNI(简称OMN)是一种基于区块链技术的数字货币。OMNI币的主要特点是通过Omni Layer协议来实现区块链上的资产管理和交易。Omni Layer 是建立在比特币区块链之上的一个协议.

Consensus Algorithm

Proof of Work 工作量证明

"它要求发起者进行一定量的运算，也就意味着需要消耗计算机一定的时间。这个概念由Cynthia Dwork 和Moni Naor 1993年在学术论文中首次提出。矿工从中选取一个区块链，并以选取最长链者为获胜者。."

Delayed Proof-of-Work 延迟工作量证明

"基于DPoW的区块链中，矿工挖矿所获得的不再是奖励的代币，而是可以焚烧的“wood”——燃木。燃烧较多wood的人或者BP或者一组BP可以获取下个事件段出块的权利，成功出块后获取奖励."

Proof of Stake 权益证明

"币的权益被用于签名一个区块，则币龄将清为零，这样必须等待至少30日才能签署另一区块。同时，为防止非常老或非常大的权益控制区块链，寻找下一区块的最大概率在90天后达到最大值，这一过程保护了网络，并随着时间逐渐生成新的币而无需消耗大量的计算能力"

Practical Byzantine Fault Tolerance 实用拜占庭容错算法

"PBFT 使用了较少（少于 20 个，之后会稍有增加）的预选定将军数，因此运行非常高效。它的优点是高交易通量和吞吐量，但是不足之处在于是中心化的，并用于许可网络。."

Proof of Elapsed Time 所用时间证明

"网络中的每位参与节点都必须等待一个随机选取的时期，首个完成设定等待时间的节点将获得一个新区块。区块链网络中的每个节点会生成一个随机的等待时间，并休眠一个设定的时间。"
